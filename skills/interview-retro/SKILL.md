# 실전 면접 회고 스킬

## 핵심 관점

**실전 면접은 가장 비싼 모의면접이다 — 받은 질문을 버리면 그 비용이 사라진다.**

- 회고의 목적은 잘잘못 채점이 아니라 **자산 축적**입니다. 실전에서 받은 질문은
  질문지 커버리지·인터뷰 노트·다음 모의면접으로 환류되어 다음 지원의 준비 비용을 낮춥니다
- 합격/불합격 결과와 무관하게 실행 가치가 있습니다 — 어느 쪽이든 받은 질문은 남습니다
- 기억은 빠르게 증발하므로 면접 직후 실행할수록 회수율이 높습니다

## 회고 수집 프로토콜

### 비난 없는 회고 톤 — 수집 전 구간 필수 준수

| 하지 않는 것 | 하는 것 |
|-------------|--------|
| 수집 중 답변에 대한 평가·교정·훈수 ("그건 이렇게 답했어야죠") | 정보만 수집하고, 분석은 갭 분석 단계에서 일괄 수행 |
| "못 답함"을 실패로 규정하는 표현 | "다음 지원 전에 채울 수 있는 갭을 발견했다"로 리프레이밍 |
| 자책성 발화("제가 부족해서…")에 동조하거나 반박 | 발화에서 사실(질문·답변·판정)만 추출하고 판단은 보류 |
| 기억나지 않는 질문·답변을 추측으로 채움 | "(기억 안 남)"을 그대로 기록 — 답변 창작 금지 |

이 원칙은 `/mock-interview`의 "세션 중 피드백 금지"와 같은 설계입니다:
수집 중 훈수가 시작되면 사용자가 기억을 꺼내는 것을 멈추고 방어를 시작해 회수율이 떨어집니다.

### 수집 순서와 단위

**한 번에 하나씩** 진행합니다 (목록을 한꺼번에 요구하지 않음):

1. **면접 단계** (택 1): 서류후 과제 / 1차 기술 / 2차 심층 / 임원·컬처핏
2. **질문 루프** — 질문마다 3단계로 수집하고 **즉시 보고서 파일에 append**:
   - 받은 질문 (기억나는 대로 — 정확하지 않아도 됨)
   - 당시 답변 요지 (요지만 — 원문 복원을 강요하지 않음)
   - 자가 판정 (아래 3단계 앵커)
3. **전체 소감** (자유 발화 — 분위기, 예상과 달랐던 점, 스스로 느낀 것)

### 자가 판정 3단계 앵커

| 판정 | 앵커 (하나라도 해당하면 그 판정) |
|------|--------------------------------|
| 잘 답함 | 준비한 구조대로 답했고, 꼬리질문에도 막히지 않음 |
| 버벅임 | 답은 했지만 구조 없이 헤맴 / 수치·근거를 대지 못함 / 꼬리질문에서 말이 꼬임 |
| 못 답함 | "모르겠습니다"로 넘어감 / 질문 의도와 다른 답을 함 / 답변을 시작하지 못함 |

판정이 애매하면 **더 나쁜 쪽을 선택**합니다 (보수 판정 —
환류 대상을 놓치는 손해가 과잉 환류의 손해보다 큽니다).

## 갭 분류 기준 3종

받은 질문마다 **A → B → C 순서로 검사하고 첫 매칭에서 확정**합니다.

| 분류 | 정의 | 판정 기준 |
|------|------|----------|
| **A. 질문지 수록** (리허설 효과 확인) | 받은 질문이 `resumes/[이름]_questionnaire.md`의 특정 질문과 같은 검증 대상을 겨냥 | 질문지의 Qn(꼬리질문 3줄 포함)과 검증 대상이 일치 — 매칭된 Qn 번호를 근거로 기록 |
| **B. 질문은행 영역 내 미수록** | 질문지에는 없지만 질문은행이 커버하는 영역에 속함 | 아래 '질문은행 영역 목록'의 어느 한 영역에 속함 — 영역명을 근거로 기록 |
| **C. 완전 신규** (커버리지 갭) | A·B 모두 아님 | 회사 도메인 특화, 최신 기술, 질문은행 밖 CS 등 — 가장 값진 수확이므로 반드시 기록 |

**매칭은 표면 문구가 아니라 검증 대상 기준**입니다:

> 받은 질문 "리액트에서 리스트가 느릴 때 어떻게 하시겠어요?"
> ↔ 질문지 Q "리렌더링 최적화 경험을 설명해주세요"
> → 검증 대상이 동일(렌더링 성능 최적화) → **A** (표현이 달라도 매칭)

### 질문은행 영역 목록 (B 판정용)

| 트랙 | 영역 | 출처 |
|------|------|------|
| 기술 | JS/TS · 프레임워크 · CSS/스타일링 · 웹 성능 · 협업/커뮤니케이션 · 브라우저/네트워크/보안 | `skills/interview-questionnaire/SKILL.md` |
| 임원(executive) | A(커리어·성장 서사) · B(사업·조직 관점) · C(책임·판단) | `skills/culture-fit-interview/SKILL.md` |
| 컬처핏(culture) | A(협업 스타일) · B(일하는 방식·가치관) · C(자기 인식) | `skills/culture-fit-interview/SKILL.md` |

### 분류별 의미와 활용

- **A**: 질문지 준비가 실전에 적중했다는 증거. 자가 판정과 교차하면 더 중요한 신호가 나옵니다 —
  **"A인데 버벅임/못 답함"은 지식 갭이 아니라 전달(리허설) 갭**입니다 (액션 아이템 매핑 참조)
- **B**: 질문지 생성이 해당 영역을 덜 뽑았다는 신호 → `/create-questionnaire` 재생성 근거
- **C**: 질문은행 자체의 사각지대 → retro 보고서에 축적해 다음 질문지 생성 시 수동 참고

**질문지가 없는 경우**: A 검사를 건너뛰고 B → C만 수행하며,
보고서 헤더에 "질문지 미존재 — 질문은행 대조만 수행"을 명기합니다.

## 환류 규칙 — 어떤 답변이 어느 자산으로 가는가

### 매핑 (고정)

| 질문 성격 | 환류 대상 자산 | 환류 방법 |
|----------|--------------|----------|
| 기술 개념 질문 (특정 프로젝트와 무관한 지식형) | 질문지 커버리지 | 질문지 파일을 직접 수정하지 않습니다 — retro 보고서의 갭 분류 표에 기록하고 `/create-questionnaire` 재생성 액션 아이템으로 연결 |
| 경험·프로젝트 질문 (특정 포트폴리오 항목과 연결) | `portfolio/<slug>/interview-notes.md` | "## 실전 환류" 섹션으로 파일 끝에 append (아래 형식) |
| 컬처핏·임원 질문 | retro 보고서 '다음 모의면접 참고' 섹션 | `/mock-interview` 재응시 액션 아이템으로 연결 (모의면접 세션 파일을 직접 수정하지 않음) |

### 환류 대상 선정 기준

| 자가 판정 | 환류 여부 |
|----------|----------|
| 못 답함 | **필수 후보** |
| 버벅임 | 선택 후보 — 사용자에게 확인 |
| 잘 답함 | 환류하지 않음 (A 분류로 리허설 효과만 보고서에 기록) |

### slug 매칭 기준 (경험 질문)

- 받은 질문이 언급하는 프로젝트·경험을 `portfolio/*/interview-notes.md` **1행의
  이력서 한 줄 원문**(`# 인터뷰 노트: …`)과 대조합니다
- 매칭되는 slug가 **없으면 append하지 않고** retro 보고서에만 기록합니다 —
  **새 slug를 만들지 않습니다** (slug 생성은 `/portfolio` Phase 1의 책임)

### interview-notes.md append 형식 — append-only 필수

- 파일 **끝에만** 추가합니다. 기존 섹션의 수정·삭제·재정렬은 절대 금지입니다
- 같은 날짜·같은 회사의 여러 질문은 하나의 "## 실전 환류" 섹션에 모아 기록합니다

```markdown
## 실전 환류 ([YYYY-MM-DD], [회사])

> /interview-retro가 추가한 실전 면접 질문입니다. 다음 /portfolio 실행 시
> Phase 2가 이 섹션을 보강 재료로 로드합니다. 삭제하지 마세요.

### Q. [실전에서 받은 질문 원문 — 기억나는 대로]
**당시 답변:** [답변 요지 그대로 — 창작 금지, 기억 안 나면 "(기억 안 남)"]
**자가 판정:** [못 답함|버벅임]
**보강 필요 포인트:** [무엇이 준비되면 답할 수 있는가 — Before/After 수치, 의사결정 근거, 실패 서사 등]
```

## 반복 취약 영역 판정 규칙

- **대조 대상**: 이전 retro 보고서들(`resumes/[이름]_retro_*.md`)의 '받은 질문 기록' + 이번 회고
- **판정 단위**: 질문 문구가 아니라 **영역** —
  기술 질문은 질문은행 6영역, 경험 질문은 portfolio slug, 컬처핏·임원 질문은 질문은행 카테고리(A/B/C)
- **판정 기준**: 같은 영역에서 자가 판정 "버벅임" 또는 "못 답함"이
  **retro 2회 이상**(이번 회고 포함)에 걸쳐 등장하면 반복 취약 영역
- 반복 취약 영역과 연결된 액션 아이템은 목록 **최상단**에 배치하고 "N회 반복" 을 표기합니다
  (한 번의 갭은 우연일 수 있지만 반복은 구조적 약점)
- 이전 retro가 없으면 이 판정을 생략하고
  "첫 회고 — 다음 회고부터 반복 취약 영역을 추적합니다"를 보고서에 명기합니다

## 액션 아이템 생성 규칙

**모든 액션 아이템은 이 플러그인의 기존 커맨드와 연결된 실행 가능 항목이어야 합니다.**
"CS 공부하기", "말하기 연습하기" 같은 커맨드 미연결 항목은 생성하지 않습니다.
각 항목에는 실행할 **커맨드 원문(인자 포함)** 을 그대로 기재합니다.

| 회고 결과 (조건) | 액션 아이템 (고정 매핑) |
|-----------------|------------------------|
| B 분류 질문 존재 | `/create-questionnaire resumes/[이름].md` 재생성 — 누락 영역이 포함되는지 확인. 작성 후 `/evaluate`로 검증 |
| C 분류 질문 존재 | `/create-questionnaire` 재생성 시 이 retro 보고서의 갭 분류 표를 함께 열어 신규 영역 질문을 수동 추가 |
| 경험 질문 환류 발생 (interview-notes append) | `/portfolio resumes/[이름].md` 재실행 — 해당 slug의 Phase 2가 환류된 노트를 로드해 보강 질문만 진행 |
| 컬처핏·임원 질문 버벅임/못 답함 | `/mock-interview resumes/[이름].md --type culture` 또는 `--type executive` 재응시 — 응시 전 retro의 '다음 모의면접 참고' 섹션으로 셀프 리허설 |
| A 분류인데 버벅임/못 답함 (준비했는데 무너짐) | 지식 갭이 아니라 전달 갭 — 질문지 답변을 다시 작성해 `/evaluate resumes/[이름]_questionnaire.md`로 재점검, 또는 `/mock-interview` 재응시로 말하기 리허설 |
| 반복 취약 영역 판정 | 위 매핑 중 해당 항목을 최상단 배치 + "N회 반복" 표기 |
