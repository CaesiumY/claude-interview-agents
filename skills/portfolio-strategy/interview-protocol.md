# Phase 2: Interview — 디테일 채우기 프로토콜

## 목적

**"면접관이 이력서를 보고 물어볼 법한 질문을 미리 풀어두자"**

빈 페이지에서 시작하는 인지 부담을 없애고, 구체적 질문으로 진짜 경험 디테일을 추출합니다.
경험의 디테일과 의사결정권은 인간에게 있습니다 — AI는 질문만 하고, 답변을 창작하지 않습니다.

## 질문 4카테고리

항목당 3~5개 질문을 선별하되, 카테고리 균형을 지킵니다:

| 카테고리 | 목적 | 질문 형태 | 필수 여부 |
|---------|------|----------|----------|
| 1. 앵커링 | 이력서에서 추론한 정보 검증 | "~로 이해했는데, 맞나요?" | 권장 1개 |
| 2. 정량 | 결과(R)를 채울 수치 확보 | "개선 전후 수치가 어떻게 되나요?" | **필수 1개 이상** |
| 3. 의사결정·트레이드오프 | 선택의 근거 확보 | "다른 선택지 대신 이걸 고른 이유는?" | 권장 1개 |
| 4. 실패 | 시도-실패-학습 서사 확보 | "시도했다가 안 된 접근이 있었나요?" | 권장 1개 |

### 질문 선별 규칙
- Phase 1의 **PSR 누락 표시가 최우선 입력**: `[R?]`이면 정량 질문 추가, `[P?]`이면 앵커링으로 문제 상황 확인
- JD 컨텍스트가 있으면 인재상 슬롯과 연결되는 의사결정 질문을 우선

## 진행 프로토콜

1. **한 번에 한 질문씩** 제시합니다 (질문 목록을 한꺼번에 던지지 않음)
2. 답변을 받으면 **즉시** `portfolio/<slug>/interview-notes.md`에 append합니다
   - 메모리에만 두지 않음 — 세션이 중단돼도 재개 가능
3. 답변이 모호하면 짧은 확인 질문 1개까지만 허용 (심문이 되지 않게)
4. "모르겠다/기억 안 남" 답변도 그대로 기록합니다 (창작 금지)
5. 모든 항목 완료 후 GATE 2: 사용자에게 노트 파일을 열어 검토·편집하게 안내

## 재사용 규칙 (핵심 자산화)

**"한 번 답한 것은 다시 답할 필요가 없어야 한다"**

- 시작 전 `portfolio/<slug>/interview-notes.md` 존재 여부를 확인
- 존재하면: 기존 답변을 로드하고, 새 JD 관점에서 **부족한 질문만** 추가로 진행
- 기존 노트는 절대 삭제·덮어쓰기하지 않고 append만 합니다

## interview-notes.md 형식

```markdown
# 인터뷰 노트: [이력서 한 줄 원문]

> 이 파일은 영구 자산입니다. 다른 회사 JD에 재사용됩니다. 삭제하지 마세요.

## 세션: YYYY-MM-DD (JD: 회사명 또는 "일반")

### Q1. [앵커링] 질문 내용
**A:** 답변 원문 그대로

### Q2. [정량] 질문 내용
**A:** 답변 원문 그대로

...
```
