# Phase 1: Curation — JD 파싱과 항목 채점

## JD 파싱 5슬롯

JD(직무 설명서)가 제공되면 아래 5개 슬롯으로 구조화합니다. 파싱 결과는 **먼저**
`portfolio/_curation/curation-[YYYY-MM-DD].md`의 "JD 요약 (5슬롯)" 절에 기록하고,
GATE 1에서 항목이 선택된 **후** 각 `portfolio/<slug>/jd-context.md`로 복사됩니다
(파싱 시점에는 slug 디렉토리가 아직 존재하지 않습니다).
JD가 없으면 슬롯을 비워두고 일반적인 프론트엔드 3년차 기준으로 채점합니다.

| 슬롯 | 내용 | 추출 예시 |
|------|------|----------|
| 1. 회사·도메인 | 회사가 속한 산업과 제품 특성 | "커머스 플랫폼, 월 100만 트래픽" |
| 2. 핵심 책임 | 이 포지션이 실제로 할 일 | "디자인 시스템 구축, 성능 개선" |
| 3. 우대 기술 키워드 | 명시된 기술 스택 (필수/우대 구분) | "Next.js(필수), Storybook(우대)" |
| 4. 시니어리티 | 기대 연차와 역할 수준 | "3-5년, 독립적 기능 리드" |
| 5. 인재상·평가 기준 | 조직이 반복적으로 강조하는 가치 | "데이터 기반 의사결정, 오너십" |

## 비정형 JD 처리

실제 채용 공고는 5슬롯이 온전한 경우가 드뭅니다. 아래 규칙으로 일관되게 처리합니다:

| 상황 | 처리 규칙 |
|------|----------|
| 슬롯에 해당 정보 없음 | 해당 슬롯에 "(JD에 명시 없음)"으로 기록. 그 슬롯과 연결된 배점은 0점 처리하지 않고 JD 없음 대체 기준으로 채점 (기술=시장 수요, 책임=3년차 기대 역할, 인재상=일반적 협업·성장 가치) |
| 필수/우대 구분 없음 | 전부 우대 기술로 간주 (필수 2배 가중 미적용) |
| 복수 포지션 혼재 | 파싱 전에 사용자에게 대상 포지션을 확인 (AskUserQuestion) |
| 영어 JD | 슬롯 내용을 한국어 요약으로 기록하되 원문 키워드는 병기 (예: "디자인 시스템 구축(design system)") |

## 가중치 기반 채점 (항목당 100점)

이력서의 각 경험 한 줄(bullet)을 아래 분해식으로 채점합니다:

| 기준 | 배점 | 판정 방법 |
|------|------|----------|
| 기술 매칭 | 40점 | 슬롯 3(우대 기술)과의 겹침. 필수 기술 매칭은 우대 기술의 2배 가중 |
| 책임 매칭 | 30점 | 슬롯 2(핵심 책임)와 경험 내용의 유사성 |
| 인재상 매칭 | 20점 | 슬롯 5(인재상)를 증명할 수 있는 경험인가 |
| Featured 항목 | 10점 | 이력서에서 상단 배치/강조된 항목인가 |

JD가 없는 경우: 기술 매칭은 시장 수요 기준, 책임 매칭은 3년차 기대 역할 기준으로 대체합니다.

## PSR 분해 검증

각 이력서 한 줄을 **문제(P)·해결(S)·결과(R)** 셋으로 분해하고 누락 요소를 표시합니다:

```
"레거시 번들 최적화로 초기 로딩 40% 개선"
→ P: [P?] 무엇이 왜 문제였는지 없음
→ S: 번들 최적화 (구체 방법 불명)
→ R: 초기 로딩 40% 개선 ✓
```

- `[P?]` 문제 상황 누락 — 인터뷰에서 "왜 필요했는지"를 물어야 함
- `[S?]` 해결 과정 누락 — 인터뷰에서 "무엇을 어떻게 했는지"를 물어야 함
- `[R?]` 정량 결과 누락 — 인터뷰에서 수치를 반드시 확보해야 함

**PSR 누락 표시는 Phase 2 질문 선별의 입력이 됩니다** — 누락된 요소를 채우는 질문을 우선 배치합니다.
