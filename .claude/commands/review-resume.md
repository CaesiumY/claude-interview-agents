# 이력서 리뷰 명령

이력서를 분석하고 프론트엔드 3년차 이직에 맞게 리뷰합니다.

## 사용법
```
/review-resume [이력서파일경로]
```

## 실행 절차

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- 지원되는 형식: .md, .txt, .pdf

### 2단계: 스킬 가이드 참조
`.claude/skills/resume-review.md` 스킬을 참조하여 평가 기준 적용

### 3단계: 이력서 분석
다음 항목을 분석합니다:

1. **기본 정보 추출**
   - 이름, 연락처, 포트폴리오/GitHub 링크
   - 경력 연수 확인 (3년차 기준)

2. **기술 스택 분석**
   - 주요 프레임워크 (React/Vue/Angular)
   - 상태 관리 도구
   - TypeScript 사용 여부
   - 테스트 도구
   - 빌드/배포 도구

3. **프로젝트 경험 평가**
   - 프로젝트별 역할과 기여도
   - 정량적 성과 포함 여부
   - 기술적 도전과 해결 과정

4. **3년차 체크포인트 확인**
   - 독립적 기능 구현 경험
   - 기술적 의사결정 경험
   - 성능 최적화 경험
   - 코드 리뷰 경험
   - 주니어 온보딩/멘토링 경험

### 4단계: 점수 산정 및 피드백 생성
- 항목별 점수 (총 100점)
- 구체적인 개선 제안
- 예상 서류 통과율

### 5단계: 결과 저장
- 결과를 `resumes/[이름]_review.md` 파일로 저장

---

## 출력 예시

```markdown
# 이력서 리뷰 결과

## 총점: 75/100

### 항목별 점수
| 항목 | 점수 | 평가 |
|------|------|------|
| 기본 구성 요소 | 16/20 | GitHub 링크 누락 |
| 기술 역량 표현 | 22/30 | TypeScript 경험 불명확 |
| 성과 중심 기술 | 18/25 | 정량적 성과 부족 |
| 프로젝트 경험 | 12/15 | 역할 명확 |
| 가독성 및 형식 | 7/10 | 2페이지 초과 |

### 강점
- React와 상태 관리(Redux) 경험이 풍부함
- 다양한 프로젝트 경험

### 개선 필요 사항
- GitHub 링크 추가 필요
- 성능 개선 수치 추가 권장
- 이력서 길이 2페이지로 압축

### 구체적 수정 제안
1. "프론트엔드 개발 담당" → "React 기반 어드민 대시보드 개발, 로딩 속도 40% 개선"
2. 기술 스택에 TypeScript 숙련도 명시
3. 각 프로젝트에 Before/After 성과 추가

### 예상 서류 통과율
- 스타트업: 70%
- 중견기업: 55%
- 대기업: 40%
```
