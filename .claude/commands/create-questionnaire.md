# 면접 질문지 생성 명령

이력서를 기반으로 맞춤형 면접 질문지를 생성합니다.

## 사용법
```
/create-questionnaire [이력서파일경로]
```

## 실행 절차

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기

### 2단계: 이력서 분석
1. **경력 분석**: 3년차 확인
2. **기술 스택 추출**: 주요 기술, 프레임워크 식별
3. **프로젝트 분석**: 핵심 프로젝트, 역할, 성과 파악
4. **특이사항 식별**: 이직 횟수, 공백기, 특별한 경험

### 3단계: 스킬 가이드 참조
`.claude/skills/interview-questionnaire.md` 스킬을 참조하여 질문 구성

### 4단계: 질문지 구성

#### 공통 질문 (기본 역량)
```markdown
## JavaScript/TypeScript 질문 (필수)
1. 클로저에 대해 설명해주세요
   → 심화: 클로저를 활용한 실제 코드 예시

2. 이벤트 루프 동작 방식을 설명해주세요
   → 심화: microtask vs macrotask

3. TypeScript Generic 사용 경험을 설명해주세요
   → 심화: Conditional Types 활용 경험
```

#### 프레임워크 질문 (이력서 기반 선택)
- React 경험자: 상태관리, Hooks, 렌더링 최적화
- Vue 경험자: Composition API, 반응성 시스템

#### 이력서 기반 개별 질문
```markdown
## [프로젝트명] 관련 질문
1. "이 프로젝트에서 [기술]을 선택한 이유는?"
2. "[성과]를 달성하기 위해 구체적으로 무엇을 했나요?"
3. "가장 어려웠던 기술적 문제와 해결 방법은?"
```

#### 협업/소프트스킬 질문
```markdown
## 협업 경험
1. 코드 리뷰에서 의견 충돌 시 어떻게 해결하셨나요?
2. 백엔드/디자이너와 협업 시 갈등 경험과 해결 방법
3. (3년차) 주니어 개발자 온보딩 경험이 있으신가요?
```

### 5단계: 연계형 질문 구조 적용
하나의 주제에 대해 깊이 파고들 수 있도록 심화 질문 추가

```
기본 → 심화 → 경험 → 상황대응
```

### 6단계: 결과 저장
- 질문지를 `resumes/[이름]_questionnaire.md` 파일로 저장

## 출력 형식

```markdown
# [이름]님 맞춤 면접 질문지

## 기본 정보
- 경력: 3년차
- 주요 기술: React, TypeScript, Redux
- 최근 프로젝트: [프로젝트명]

---

## 1. 기술 질문 (40분)

### JavaScript/TypeScript (15분)

**Q1. 클로저(Closure)란 무엇인가요?**
- 예상 답변 키워드: 렉시컬 스코프, 외부 변수 참조, 메모리
- 심화: "클로저로 인한 메모리 누수 경험이 있나요?"

**Q2. 이벤트 루프를 설명해주세요**
- 예상 답변 키워드: 콜스택, 태스크 큐, 렌더링
- 심화: "Promise와 setTimeout의 실행 순서는?"

### React (15분)

**Q3. useMemo와 useCallback의 차이점은?**
- 예상 답변 키워드: 메모이제이션, 참조 동일성
- 심화: "과도한 메모이제이션의 단점은?"

**Q4. 전역 상태 관리 도구를 선택할 때 기준은?**
- 예상 답변 키워드: 프로젝트 규모, 러닝커브, 보일러플레이트

### 성능 최적화 (10분)

**Q5. 웹 성능 최적화 경험을 설명해주세요**
- 예상 답변 키워드: Core Web Vitals, 번들 사이즈, 레이지 로딩

---

## 2. 이력서 기반 질문 (15분)

**Q6. [프로젝트명]에서 Redux를 선택한 이유는?**
- 확인 포인트: 기술 선택 근거, 대안 검토 여부

**Q7. "[성과]" 달성을 위해 구체적으로 무엇을 했나요?**
- 확인 포인트: 본인 기여도, 구체적 행동

**Q8. 해당 프로젝트에서 가장 어려웠던 문제는?**
- 확인 포인트: 문제 해결 과정, 학습 능력

---

## 3. 협업/소프트스킬 (10분)

**Q9. 코드 리뷰에서 의견 충돌 시 어떻게 해결하셨나요?**
- 평가 포인트: 커뮤니케이션, 논리적 설득

**Q10. 주니어 개발자 온보딩 경험이 있으신가요?**
- 평가 포인트: 리더십, 지식 공유

---

## 4. 역질문 시간 (5분)
> 지원자의 관심사와 가치관 파악
```
