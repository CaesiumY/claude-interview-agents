# 면접 질문지 생성 명령

이력서를 기반으로 맞춤형 면접 질문지를 생성합니다.

## 사용법
```
/create-questionnaire [이력서파일경로]
```

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- `portfolio/*/final.md`가 존재하면 함께 읽기 (질문 소스로 활용)

**예외 처리**:
- **파일 미발견**: 이력서 파일이 없거나 경로가 틀리면 `resumes/` 폴더의 파일 목록을 보여주고
  사용자에게 경로 확인을 요청한 뒤 **중단**합니다. 임의로 추측해 진행하지 않습니다.
- **[이름] 결정 규칙**: 산출물 파일명의 `[이름]`은 입력 이력서 파일의
  **베이스네임(확장자 제외)을 그대로** 사용합니다 (예: `resumes/kim.md` →
  `resumes/kim_questionnaire.md`). `/review-resume`의 `[이름]_review.md` /
  `[이름]_evaluation.md` / `[이름]_mock_[type]_[YYYY-MM-DD].md` 파일 패밀리와
  이름이 일치해야 `/evaluate`의 이력서·모의면접 평가 파일 탐색이 끊어지지 않습니다.
  이력서 본문에서 추출한 지원자 이름은 **문서 제목 표시용으로만** 사용합니다
  (예: 질문지 제목 `# ○○○님 맞춤 면접 질문지`) — 본문에 이름이 없으면 제목에도
  베이스네임을 사용합니다. 파일명에는 본문 추출 이름을 쓰지 않습니다.

### 2단계: 이력서 분석
1. **경력 분석**: 연차 확인 (기본 대상: 3년차 = 2~4년)
2. **기술 스택 추출**: 주요 기술, 프레임워크 식별
3. **프로젝트 분석**: **핵심 프로젝트 2~3개 선정**, 역할·성과 파악 — 이력서 기반 질문의 생성 단위
4. **특이사항 식별**: 이직 횟수, 공백기, 특별한 경험

**예외 처리**:
- **연차 불일치**: 분석 결과 2~4년차가 아니면 사용자에게 고지하고, 해당 연차에 맞게
  난이도 라벨 비율을 조정해 진행합니다. 조용히 3년차 기준으로 생성하지 않습니다.
  - 1~2년차: [스크리닝] 60% 이상 (기본기 정확성 위주)
  - 5년차 이상: [본선] 70% 이상 + 설계/아키텍처 질문 추가
- **프레임워크 판별**:
  - React와 Vue가 모두 있으면 **최근/주력 프로젝트에서 사용한 쪽**을 우선합니다
  - React/Vue 외 프레임워크(Svelte, Angular, 바닐라 JS 등)가 주력이면, 질문은행의
    개념 축(상태 관리·성능 최적화·컴포넌트 설계)을 해당 프레임워크 용어로 변형해 생성합니다

### 3단계: 스킬 가이드 참조
`${CLAUDE_PLUGIN_ROOT}/skills/interview-questionnaire/SKILL.md` 스킬을 참조하여 질문 구성

### 4단계: 질문지 구성

**정량 규칙 — 매 실행에서 동일하게 적용합니다**:

| 규칙 | 값 |
|------|-----|
| 총 질문 수 | **20~25개 고정** (역질문 3개 포함) |
| 공통 질문은행 | 10~12개 |
| 이력서 기반 | 핵심 프로젝트 2~3개 × 프로젝트당 4개 (4카테고리 각 1개) |
| 협업/소프트스킬 | 2~3개 |
| 역질문 | 3개 |
| 난이도 비율 | [스크리닝] 40% : [본선] 60% (연차 예외 시 2단계 규칙 적용) |

- 합계가 25개를 넘으면 **공통 질문은행에서 줄여** 맞춥니다. 이력서 기반 질문은
  '내 이야기'로 답할 수 있는 차별화 핵심이므로 줄이지 않습니다.
- **모든 질문에 난이도 라벨을 부착**합니다:
  - **[스크리닝]** — 개념 정의·동작 원리 확인 (정확성 검증, 1차 화상면접 대비)
  - **[본선]** — 본인 경험·트레이드오프·수치 검증 (깊이 검증, 최종 기술면접 대비)

#### 공통 질문 (기본 역량)
- JavaScript/TypeScript 필수 질문 (클로저, 이벤트 루프, 비동기, TypeScript)
- 프레임워크 질문 (이력서 기반 React/Vue 선택 — 판별 규칙은 2단계 참조)
- CSS/웹 성능 질문
- 브라우저/네트워크/보안 질문 — **이력서에 관련 언급이 없어도 반드시 포함**
  (본선 빈출이지만 준비 사각지대가 되기 쉬운 영역)

#### 이력서 기반 개별 질문 (4카테고리 적용)
각 핵심 프로젝트마다 4카테고리로 질문을 1개씩 생성하고 카테고리를 표기합니다:

| 카테고리 | 질문 형태 |
|---------|----------|
| [앵커링] | "○○을 주도적으로 도입하신 것으로 이해했는데, 맞나요?" |
| [정량] | "[성과]의 개선 전후 수치와 측정 방법은?" — **프로젝트당 최소 1개 필수** |
| [의사결정] | "[기술]을 대안과 비교했을 때 무엇을 포기하고 무엇을 얻었나요?" |
| [실패] | "시도했다가 되돌린 결정이 있나요?" |

**포트폴리오가 있는 경우**: `portfolio/<slug>/final.md`에 심어진 질문 트리거
(트레이드오프, 실패 서사, 특이한 수치)에서 질문을 우선 생성합니다.

#### 협업/소프트스킬 질문
- 코드 리뷰, 협업 갈등, 주니어 온보딩 경험

#### 역질문
- 스킬 가이드의 '역질문 생성 규칙'에 따라 이력서 맥락과 연결된 3개 생성
  (기술 문화 검증형 / 팀 상황 파악형 / 성장 연결형 각 1개)
- 지원 회사·JD 정보가 있으면 스킬 가이드의 'JD·지원 회사 연결 규칙'을 적용해
  JD 슬롯(핵심 책임·회사·도메인·인재상)과 연결된 역질문으로 생성
- 생성 후 스킬 가이드의 **'피해야 할 역질문' 표**(처우 선공·검색 가능·범용·부정 전제·
  대상 미스매치)에 해당하는 질문이 없는지 자체 점검 — 해당하면 교체
- 이 역질문 3개는 `/mock-interview`의 역질문 턴에서 리허설할 수 있는 재사용 자산입니다

### 5단계: 연계형 질문 구조 적용
공통 질문은행에서 가져온 질문은 하나의 주제를 깊이 파고들 수 있도록 4단계 구조로 전개합니다:

```
기본 → 심화 → 경험 연결 → 상황 대응
```

이력서 기반 질문은 4카테고리가 이미 이 역할을 하므로 4단계 전개를 생략합니다.

### 6단계: 결과 저장
- 질문지를 `resumes/[이름]_questionnaire.md` 파일로 저장 (UTF-8)
- 각 질문 블록 바로 아래에 답변란 포함, 예상 답변 포인트는 파일 말미 '답변 가이드'로 분리
  (스킬 가이드의 출력 형식 참조)
- **동일 경로에 파일이 이미 존재하면** 덮어쓸지 사용자에게 먼저 확인합니다.
  기존 파일을 보존하려면 `resumes/[이름]_questionnaire_[YYYY-MM-DD].md`로 저장을 제안합니다.

> **다음 단계 안내**: 질문지에 답변을 작성한 뒤 `/evaluate resumes/[이름]_questionnaire.md`로 평가를 받으세요.
> 임원/컬처핏 면접 대비는 `/mock-interview`를 사용하세요.
