# 면접 질문지 생성 명령

이력서를 기반으로 맞춤형 면접 질문지를 생성합니다.

## 사용법
```
/create-questionnaire [이력서파일경로]
```

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- `portfolio/*/final.md`가 존재하면 함께 읽기 (질문 소스로 활용)

### 2단계: 이력서 분석
1. **경력 분석**: 3년차 확인
2. **기술 스택 추출**: 주요 기술, 프레임워크 식별
3. **프로젝트 분석**: 핵심 프로젝트, 역할, 성과 파악
4. **특이사항 식별**: 이직 횟수, 공백기, 특별한 경험

### 3단계: 스킬 가이드 참조
`${CLAUDE_PLUGIN_ROOT}/skills/interview-questionnaire/SKILL.md` 스킬을 참조하여 질문 구성

### 4단계: 질문지 구성

#### 공통 질문 (기본 역량)
- JavaScript/TypeScript 필수 질문 (클로저, 이벤트 루프, 비동기, TypeScript)
- 프레임워크 질문 (이력서 기반 React/Vue 선택)
- CSS/웹 성능 질문

#### 이력서 기반 개별 질문 (4카테고리 적용)
각 프로젝트/경험마다 4카테고리로 질문을 생성하고 카테고리를 표기합니다:

| 카테고리 | 질문 형태 |
|---------|----------|
| [앵커링] | "○○을 주도적으로 도입하신 것으로 이해했는데, 맞나요?" |
| [정량] | "[성과]의 개선 전후 수치와 측정 방법은?" — **항목당 최소 1개 필수** |
| [의사결정] | "[기술]을 대안과 비교했을 때 무엇을 포기하고 무엇을 얻었나요?" |
| [실패] | "시도했다가 되돌린 결정이 있나요?" |

**포트폴리오가 있는 경우**: `portfolio/<slug>/final.md`에 심어진 질문 트리거
(트레이드오프, 실패 서사, 특이한 수치)에서 질문을 우선 생성합니다.

#### 협업/소프트스킬 질문
- 코드 리뷰, 협업 갈등, 주니어 온보딩 경험

### 5단계: 연계형 질문 구조 적용
하나의 주제에 대해 깊이 파고들 수 있도록 심화 질문 추가

```
기본 → 심화 → 경험 → 상황대응
```

### 6단계: 결과 저장
- 질문지를 `resumes/[이름]_questionnaire.md` 파일로 저장 (UTF-8)
- 각 질문에 답변 기록란 포함 (스킬 가이드의 출력 형식 참조)

> **다음 단계 안내**: 질문지에 답변을 작성한 뒤 `/evaluate resumes/[이름]_questionnaire.md`로 평가를 받으세요.
> 임원/컬처핏 면접 대비는 `/mock-interview`를 사용하세요.
