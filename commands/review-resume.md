# 이력서 리뷰 명령

이력서를 분석하고 프론트엔드 3년차 이직에 맞게 리뷰합니다.

## 사용법
```
/review-resume [이력서파일경로]
```

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- 지원되는 형식: .md, .txt, .pdf

### 2단계: 스킬 가이드 참조
`${CLAUDE_PLUGIN_ROOT}/skills/resume-review/SKILL.md` 스킬을 참조하여 평가 기준 적용

### 3단계: 이력서 분석
다음 항목을 분석합니다:

1. **기본 정보 추출**
   - 이름, 연락처, 포트폴리오/GitHub 링크
   - 경력 연수 확인 (3년차 기준)

2. **기술 스택 분석**
   - 주요 프레임워크 (React/Vue/Angular)
   - 상태 관리 도구
   - TypeScript 사용 여부
   - 테스트 도구
   - 빌드/배포 도구

3. **프로젝트 경험 평가**
   - 프로젝트별 역할과 기여도
   - 정량적 성과 포함 여부
   - 기술적 도전과 해결 과정

4. **PSR 분해 검증**
   - 경험 한 줄(bullet)마다 문제(P)·해결(S)·결과(R)로 분해
   - 누락 요소를 `[P?]`/`[S?]`/`[R?]`로 표시 (스킬 가이드의 표 형식)
   - 성과 중심 기술(25점) 채점과 연동

5. **3년차 체크포인트 확인**
   - 독립적 기능 구현 경험
   - 기술적 의사결정 경험
   - 성능 최적화 경험
   - 코드 리뷰 경험
   - 주니어 온보딩/멘토링 경험

### 4단계: 점수 산정 및 피드백 생성
- 항목별 점수 (총 100점)
- 구체적인 개선 제안
- **포트폴리오 전개 추천**: PSR이 온전하거나 보강 가치가 높은 항목을 `/portfolio` 입력 후보로 추천
- 예상 서류 통과율

### 5단계: 결과 저장
- 결과를 `resumes/[이름]_review.md` 파일로 저장 (UTF-8)

## 출력 예시

```markdown
# 이력서 리뷰 결과

## 총점: 75/100

### 항목별 점수
| 항목 | 점수 | 평가 |
|------|------|------|
| 기본 구성 요소 | 16/20 | GitHub 링크 누락 |
| 기술 역량 표현 | 22/30 | TypeScript 경험 불명확 |
| 성과 중심 기술 | 18/25 | 정량적 성과 부족 (PSR: R 누락 2건) |
| 프로젝트 경험 | 12/15 | 역할 명확 |
| 가독성 및 형식 | 7/10 | 2페이지 초과 |

### PSR 분해 검증
| 이력서 한 줄 | P | S | R | 누락 |
|-------------|---|---|---|------|
| "번들 최적화로 로딩 40% 개선" | ✗ | △ | ✓ | [P?] 문제 배경, [S?] 구체 방법 |

### 강점
- React와 상태 관리(Redux) 경험이 풍부함

### 개선 필요 사항
- GitHub 링크 추가 필요
- 성능 개선 수치 추가 권장

### 구체적 수정 제안
1. "프론트엔드 개발 담당" → "React 기반 어드민 대시보드 개발, 로딩 속도 40% 개선"
2. 기술 스택에 TypeScript 숙련도 명시

### 포트폴리오 전개 추천 항목
1. "번들 최적화로 로딩 40% 개선" — R이 확보되어 있어 P/S만 인터뷰로 채우면 강력한 섹션 가능

### 예상 서류 통과율
- 스타트업: 70% / 중견기업: 55% / 대기업: 40%
```
