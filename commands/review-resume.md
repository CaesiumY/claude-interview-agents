# 이력서 리뷰 명령

이력서를 분석하고 프론트엔드 3년차 이직에 맞게 리뷰합니다.

## 사용법
```
/review-resume [이력서파일경로] [JD파일경로]
```

- `JD파일경로`(선택): 지원할 채용공고(JD) 파일을 함께 제공하면 리뷰 결과에 **JD 적합도 갭 분석** 섹션(5슬롯 요약·필수/우대 키워드 매칭표·누락 키워드·JD 기준 통과율)이 추가됩니다. 미제공 시 기존 리뷰만 수행합니다. 회사별로 JD만 바꿔 재실행하면 같은 이력서의 회사 간 적합도 비교(지원 우선순위 판단)가 가능합니다.

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- 지원되는 형식: .md, .txt, .pdf
- JD 파일 경로가 제공된 경우 JD 파일도 함께 읽기
  - JD는 `${CLAUDE_PLUGIN_ROOT}/skills/portfolio-strategy/jd-parsing.md`의 **5슬롯 파싱**
    (회사·도메인/핵심 책임/우대 기술/시니어리티/인재상)과 **비정형 JD 처리 규칙**을 그대로
    적용해 구조화합니다 — `/portfolio`와 동일 기준이어야 같은 JD에서 같은 슬롯이 나와
    회사 간 비교가 성립합니다 (별도 파싱 규칙을 만들지 않습니다)

**입력 예외 처리** (해당하는 경우 반드시 아래 분기를 따릅니다):
1. **인자 생략 또는 파일 미존재**: `resumes/` 폴더의 이력서 후보 목록(`*_review.md`, `*_questionnaire.md`, `*_evaluation.md`, `*_mock_*.md` 등 산출물 파일 제외)을 제시하고 사용자가 선택한 뒤 진행합니다. 후보가 없으면 이력서 파일 경로를 요청합니다.
2. **PDF 텍스트 추출 실패**(이미지 기반 PDF 등): 리뷰를 진행하지 말고, 이력서를 .md 또는 .txt로 변환한 뒤 다시 실행하도록 안내합니다.
3. **경력 연수 불일치**: 추출한 경력 연수가 3년차 ±1(2~4년)을 벗어나면(신입, 5년차 이상 등) 사용자에게 알리고 3년차 기준 적용 여부를 확인한 후 진행합니다. 그대로 적용한 경우 리뷰 결과 상단에 "3년차 기준 적용"임을 명시합니다.

### 2단계: resume-reviewer 서브 에이전트 호출

메인 루프는 채점을 **직접 수행하지 않고** `resume-reviewer` 서브 에이전트에 위임합니다.
채점(항목별 앵커·PSR 감점식·통과율 밴드)은 세션 모델에 좌우되면 재현성이 흔들리므로,
`model` 핀이 고정된 이 에이전트가 "같은 이력서 = 같은 점수"를 모델 수준에서 보장합니다.

```
Use the resume-reviewer subagent to score this resume:

## 평가 대상
- 이력서: [1단계에서 읽은 이력서 전문 — 또는 절대 경로(에이전트가 Read로 열람)]
- 채점 기준: ${CLAUDE_PLUGIN_ROOT}/skills/resume-review/SKILL.md
- 연차 확인 결과: [예: "3년차(추출값 3년)" / "3년차 기준 적용(추출값 5년, 사용자 승인)"]

## JD 컨텍스트 (제공된 경우에만)
- JD 5슬롯: [1단계 파싱 결과 — 우대 기술은 필수(자격요건)/우대 구분 포함]

## 요청
SKILL.md의 채점 앵커·PSR 기호 판정·성과 중심 감점식·통과율 밴드를 그대로 적용해 채점하고,
SKILL.md '출력 형식'의 마크다운 보고서 전문을 반환하세요.
- 수치 창작 금지: 원문에 없는 정량 수치는 `[X%]` 플레이스홀더 + "(실제 수치 확인 후 기입)"으로 표기
- 구체적 수정 제안은 Before → After 최소 3건
- 포트폴리오 전개 추천은 항목마다 ① 원문 bullet 따옴표 인용 ② PSR 상태·보강 포인트 ③ 사유 1줄 포함
- 예상 서류 통과율/JD 기준 통과율은 총점 연동 밴드로만 산정하고 단서 문구를 포함
- JD 컨텍스트가 없으면 'JD 적합도 갭 분석' 섹션을 넣지 않음
- 반환 전 5개 영역 점수 합 = 총점 자체 검증
```

**주의**: 같은 이름의 이전 리뷰 파일(`resumes/[이름]_review.md`)은 에이전트에 **전달하지 않습니다**
(독립 평가 원칙 유지 — 이전 점수에 앵커링되지 않도록).

### 3단계: 반환물 유효성 검사

1. 반환된 마크다운에 `## 총점: XX/100`과 5행 항목별 점수표(기본 구성 요소 / 기술 역량 표현 /
   성과 중심 기술 / 프로젝트 경험 / 가독성 및 형식)가 모두 존재하는지 확인합니다.
2. 총점 또는 5개 영역 점수 중 하나라도 누락되었거나, **5개 영역 점수의 합이 총점과 불일치**하면
   **1회 재호출**합니다. JD를 제공했는데 'JD 적합도 갭 분석' 섹션이 없는 경우에도 재호출합니다.
3. 재호출도 실패하면 사용자에게 원인을 알리고 **중단**합니다 — 반쪽짜리 리뷰를 저장하지 않습니다.

### 4단계: 결과 저장
- 3단계를 통과한 **에이전트 반환 마크다운 보고서를 그대로** `resumes/[이름]_review.md` 파일로 저장합니다 (UTF-8). 메인 루프가 내용을 다시 채점하거나 재작성하지 않습니다.
- **[이름] 결정 규칙**: 입력 이력서 파일의 **베이스네임(확장자 제외)을 그대로** 사용합니다 (예: `resumes/kim.md` → `resumes/kim_review.md`). 이후 `[이름]_questionnaire.md` / `[이름]_evaluation.md` / `[이름]_mock_[type]_[YYYY-MM-DD].md` 파일 패밀리와 이름이 일치해야 합니다.
- 부분합=총점 자체 검증은 3단계(반환물 유효성 검사)에서 이미 수행합니다 (스킬 가이드의 출력 형식 참조).

> **다음 단계 안내** (저장 후 사용자에게 출력):
> - 포트폴리오 전개 추천 항목이 있으면 `/portfolio resumes/[이름].md` (입력한 이력서 파일 경로)
> - 면접 준비는 `/create-questionnaire resumes/[이름].md`

## 출력 형식

산출물의 섹션 구성과 템플릿은 `${CLAUDE_PLUGIN_ROOT}/skills/resume-review/SKILL.md`의 **출력 형식**을 단일 기준으로 따릅니다.

## 서브 에이전트 구성

이 커맨드는 플러그인의 서브 에이전트를 활용합니다:

| 에이전트 | 역할 | 산출 |
|----------|------|------|
| `resume-reviewer` | 채점 앵커·PSR 감점식·통과율 밴드를 그대로 적용하는 이력서 채점 (model 핀으로 재현성 보장) | SKILL.md '출력 형식'의 마크다운 리뷰 보고서 전문 |

메인 루프는 오케스트레이터로서 입력 검증·JD 파싱·연차 확인·반환물 검증·파일 저장을 담당하고,
점수 산정과 피드백 생성은 전적으로 `resume-reviewer`에 위임합니다.
