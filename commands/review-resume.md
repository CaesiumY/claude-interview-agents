# 이력서 리뷰 명령

이력서를 분석하고 프론트엔드 3년차 이직에 맞게 리뷰합니다.

## 사용법
```
/review-resume [이력서파일경로] [JD파일경로]
```

- `JD파일경로`(선택): 지원할 채용공고(JD) 파일을 함께 제공하면 리뷰 결과에 **JD 적합도 갭 분석** 섹션(5슬롯 요약·필수/우대 키워드 매칭표·누락 키워드·JD 기준 통과율)이 추가됩니다. 미제공 시 기존 리뷰만 수행합니다. 회사별로 JD만 바꿔 재실행하면 같은 이력서의 회사 간 적합도 비교(지원 우선순위 판단)가 가능합니다.

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 이력서 파일 읽기
- `resumes/` 폴더 또는 지정된 경로에서 이력서 파일 읽기
- 지원되는 형식: .md, .txt, .pdf
- JD 파일 경로가 제공된 경우 JD 파일도 함께 읽기
  - JD는 `${CLAUDE_PLUGIN_ROOT}/skills/portfolio-strategy/jd-parsing.md`의 **5슬롯 파싱**
    (회사·도메인/핵심 책임/우대 기술/시니어리티/인재상)과 **비정형 JD 처리 규칙**을 그대로
    적용해 구조화합니다 — `/portfolio`와 동일 기준이어야 같은 JD에서 같은 슬롯이 나와
    회사 간 비교가 성립합니다 (별도 파싱 규칙을 만들지 않습니다)

**입력 예외 처리** (해당하는 경우 반드시 아래 분기를 따릅니다):
1. **인자 생략 또는 파일 미존재**: `resumes/` 폴더의 이력서 후보 목록(`*_review.md`, `*_questionnaire.md`, `*_evaluation.md`, `*_mock_*.md` 등 산출물 파일 제외)을 제시하고 사용자가 선택한 뒤 진행합니다. 후보가 없으면 이력서 파일 경로를 요청합니다.
2. **PDF 텍스트 추출 실패**(이미지 기반 PDF 등): 리뷰를 진행하지 말고, 이력서를 .md 또는 .txt로 변환한 뒤 다시 실행하도록 안내합니다.
3. **경력 연수 불일치**: 추출한 경력 연수가 3년차 ±1(2~4년)을 벗어나면(신입, 5년차 이상 등) 사용자에게 알리고 3년차 기준 적용 여부를 확인한 후 진행합니다. 그대로 적용한 경우 리뷰 결과 상단에 "3년차 기준 적용"임을 명시합니다.

### 2단계: 스킬 가이드 참조
`${CLAUDE_PLUGIN_ROOT}/skills/resume-review/SKILL.md` 스킬을 참조하여 평가 기준 적용

### 3단계: 이력서 분석
다음 항목을 분석합니다:

1. **기본 정보 추출**
   - 이름, 연락처, 포트폴리오/GitHub 링크
   - 경력 연수 확인 (3년차 기준)

2. **기술 스택 분석**
   - 주요 프레임워크 (React/Vue/Angular)
   - 상태 관리 도구
   - TypeScript 사용 여부
   - 테스트 도구
   - 빌드/배포 도구

3. **프로젝트 경험 평가**
   - 프로젝트별 역할과 기여도
   - 정량적 성과 포함 여부
   - 기술적 도전과 해결 과정

4. **PSR 분해 검증**
   - 경험 한 줄(bullet)마다 문제(P)·해결(S)·결과(R)로 분해
   - ✓/△/✗ 판정은 스킬 가이드의 **기호 판정 기준표**를 그대로 적용
   - 누락 요소를 `[P?]`/`[S?]`/`[R?]`로 표시 (스킬 가이드의 표 형식)
   - 성과 중심 기술(25점) 채점과 연동 (스킬 가이드의 감점식 공식 적용)

5. **3년차 체크포인트 확인**
   - 독립적 기능 구현 경험
   - 기술적 의사결정 경험
   - 성능 최적화 경험
   - 코드 리뷰 경험
   - 주니어 온보딩/멘토링 경험

### 4단계: 점수 산정 및 피드백 생성
- 항목별 점수 (총 100점) — 스킬 가이드의 **항목별 배점·채점 앵커**를 그대로 적용
- 구체적인 개선 제안 — 스킬 가이드의 **구체적 수정 제안 규칙** 준수 (원문에 없는 수치 창작 금지, `[X%]` 플레이스홀더 사용)
- **포트폴리오 전개 추천**: PSR이 온전하거나 보강 가치가 높은 항목을 `/portfolio` 입력 후보로 추천. 추천 항목마다 다음 3가지를 반드시 포함:
  1. 이력서 원문 bullet을 따옴표로 **원문 그대로** 인용
  2. PSR 상태(예: `P✗ / S△ / R✓`)와 보강 포인트
  3. 추천 사유 1줄
- 예상 서류 통과율 — 스킬 가이드의 **총점 연동 고정 매핑 밴드**로만 산정 (임의 수치 생성 금지, 단서 문구 포함)
- **JD 적합도 갭 분석** (JD 파일이 제공된 경우에만 추가 출력) — 스킬 가이드의
  'JD 적합도 갭 분석' 규칙을 그대로 적용:
  - JD 5슬롯 요약 (1단계에서 파싱한 결과)
  - 필수(자격요건)/우대 기술 키워드 vs 이력서 표기 매칭표 (일치 / 표기 불일치 / 누락)
  - 이력서에 누락된 키워드 목록과 보강 우선순위 (필수 누락 우선)
  - 표기 불일치 지적 (예: 이력서 '리액트' ↔ JD 'React' → 영문 공식 표기로 통일 권장)
  - **JD 기준 통과율**: 총점 연동 밴드에 필수 키워드 누락 보정을 적용 (스킬 가이드의
    보정 규칙 — 임의 수치 생성 금지)

### 5단계: 결과 저장
- 결과를 `resumes/[이름]_review.md` 파일로 저장 (UTF-8)
- **[이름] 결정 규칙**: 입력 이력서 파일의 **베이스네임(확장자 제외)을 그대로** 사용합니다 (예: `resumes/kim.md` → `resumes/kim_review.md`). 이후 `[이름]_questionnaire.md` / `[이름]_evaluation.md` / `[이름]_mock_[type]_[YYYY-MM-DD].md` 파일 패밀리와 이름이 일치해야 합니다.
- 저장 전 항목별 점수 합계가 총점과 일치하는지 자체 검증합니다 (스킬 가이드의 출력 형식 참조)

> **다음 단계 안내** (저장 후 사용자에게 출력):
> - 포트폴리오 전개 추천 항목이 있으면 `/portfolio resumes/[이름].md` (입력한 이력서 파일 경로)
> - 면접 준비는 `/create-questionnaire resumes/[이름].md`
