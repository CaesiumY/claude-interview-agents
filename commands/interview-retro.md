# 실전 면접 회고 명령

실전 면접에서 받은 질문을 기억이 증발하기 전에 기록하고, 기존 준비 자산
(질문지·질문은행·인터뷰 노트)과 대조해 커버리지 갭을 찾아 다음 지원의 자산으로 환류합니다.
서브 에이전트 호출 없이 메인 루프가 직접 진행합니다.

## 사용법
```
/interview-retro resumes/[이름].md --company 회사명   # 회사명 지정
/interview-retro resumes/[이름].md                    # 회사명 미지정 시 사용자에게 한 번 확인
```

> **설계 사유**: 이 커맨드는 인터랙티브 세션(질문 수집)이 중심이라 서브에이전트 없이
> 메인 루프가 직접 진행합니다 (서브에이전트는 사용자와 대화할 수 없음).

---

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.
> 시작 전에 `${CLAUDE_PLUGIN_ROOT}/skills/interview-retro/SKILL.md`를 읽어
> 회고 수집 톤, 자가 판정 앵커, 갭 분류 기준, 환류 규칙, 보고서 출력 형식을 확인하세요.

### 1단계: 준비 — 기존 자산 로드

1. 이력서 파일 읽기
2. **[이름] 결정 규칙**: 산출물 파일명의 `[이름]`은 이력서 파일의 **베이스네임(확장자 제외)을
   그대로** 사용합니다 — `/create-questionnaire`의 '[이름] 결정 규칙'과 동일합니다.
   이름이 일치해야 `resumes/[이름]_questionnaire.md` 탐색과
   이전 retro 파일(`resumes/[이름]_retro_*.md`) 대조가 끊어지지 않습니다
3. **회사명 확정**: `--company`가 없으면 AskUserQuestion으로 **한 번** 확인합니다.
   파일명에 "미지정"을 넣지 않습니다. 회사명의 공백은 파일명에서 하이픈으로 치환합니다
   (예: "토스 페이먼츠" → `toss-페이먼츠`가 아니라 `토스-페이먼츠`)
4. 기존 자산을 Glob으로 탐색해 **있는 것만** 로드합니다:
   - `resumes/[이름]_questionnaire.md` — 갭 분류 A(질문지 수록) 대조용
   - `portfolio/*/interview-notes.md` — 환류 slug 매칭용 (각 파일 1행의 이력서 한 줄 원문 확인)
   - `resumes/[이름]_retro_*.md` — 반복 취약 영역 대조용 (이전 회고)
   - 질문은행 (갭 분류 B 대조용):
     - `${CLAUDE_PLUGIN_ROOT}/skills/interview-questionnaire/SKILL.md` (기술·협업 6영역)
     - `${CLAUDE_PLUGIN_ROOT}/skills/culture-fit-interview/SKILL.md` (임원·컬처핏)

**예외 처리**:
- **이력서 파일 미발견**: `resumes/` 폴더의 파일 목록을 보여주고 경로 확인을 요청한 뒤
  **중단**합니다. 임의로 추측해 진행하지 않습니다
- **질문지 미존재**: "질문지가 없어 갭 분석을 질문은행 대조로만 수행합니다 (A 분류 생략)"를
  회고 시작 **전에** 고지하고 진행합니다. 조용히 생략하지 않습니다
- **동일 경로에 retro 파일이 이미 존재** (같은 회사·같은 날짜 재실행): 기존 파일을 덮어쓰지 않고
  파일명 끝에 `_2`, `_3` … 순번을 붙입니다

5. 보고서 파일 생성: `resumes/[이름]_retro_[회사]_[YYYY-MM-DD].md` (UTF-8) —
   스킬의 출력 형식에 따라 **헤더만 먼저** 작성합니다 (수집 중 append, 세션 중단 대비)

### 2단계: 회고 수집 (인터랙티브)

스킬의 '회고 수집 프로토콜'을 준수합니다 — **비난 없는 회고 톤**:
수집 중에는 답변에 대한 평가·교정·훈수를 하지 않고 정보만 수집합니다.
분석은 3단계에서 일괄 수행합니다.

1. AskUserQuestion으로 **면접 단계**를 수집합니다:
   서류후 과제 / 1차 기술 / 2차 심층 / 임원·컬처핏
2. **질문 루프 — 한 번에 하나씩** (목록을 한꺼번에 요구하지 않음):
   1. "기억나는 질문을 하나 알려주세요 (기억나는 대로, 정확하지 않아도 됩니다)"
   2. 질문을 받으면 → "그때 어떻게 답하셨나요? (요지만)" — 답변 요지 수집
   3. AskUserQuestion으로 **자가 판정** 수집: 잘 답함 / 버벅임 / 못 답함
      (선택지 description에 스킬의 판정 앵커를 표기 — 애매하면 더 나쁜 쪽 선택 안내)
   4. 질문·답변 요지·판정을 **즉시 보고서 파일에 append**합니다
      ("기억 안 남"도 그대로 기록 — 답변·질문을 창작하지 않습니다)
   5. "다음 질문도 기억나시나요?" — "없어요/그만/끝"이면 루프를 종료합니다
3. **전체 소감**을 수집합니다 (자유 발화 — 분위기, 예상과 달랐던 점) → 보고서에 그대로 기록

### 3단계: 갭 분석

스킬의 '갭 분류 기준 3종'에 따라 받은 질문마다 **A → B → C 순서로 검사, 첫 매칭에서 확정**합니다:

| 분류 | 의미 |
|------|------|
| A | 질문지에 있었음 — 리허설 효과 확인 |
| B | 질문은행 영역이지만 질문지에 미포함 |
| C | 완전 신규 — 커버리지 갭 |

- 매칭은 표면 문구가 아니라 **검증 대상** 기준입니다 (스킬의 매칭 예시 참조).
  매칭 근거(질문지 Qn 번호 / 질문은행 영역명 / 신규)를 분류마다 기록합니다
- 질문지 미존재 시 A를 생략하고 B → C만 수행합니다 (1단계에서 고지한 대로)
- 갭 분류 표와 리허설 적중률(질문지가 있는 경우만)을 산출해 보고서에 기록합니다

### 4단계: 환류 (append-only — 기존 내용 수정·삭제 절대 금지)

스킬의 '환류 규칙' 매핑(기술 질문→질문지 커버리지 / 경험 질문→interview-notes /
컬처핏·임원 질문→다음 모의면접 참고)에 따라:

1. **환류 후보 선정**: 자가 판정 "못 답함"은 필수 후보, "버벅임"은 선택 후보입니다
2. 경험·프로젝트 질문은 **slug 매칭**: 질문이 언급하는 경험을
   `portfolio/*/interview-notes.md` 1행의 이력서 한 줄 원문과 대조합니다
3. 매칭된 후보를 표로 제시하고 AskUserQuestion(multiSelect)으로 **append할 항목을 확정**합니다:

   | 받은 질문 | 자가 판정 | 환류 대상 |
   |----------|----------|----------|
   | (질문 요약) | 못 답함 | portfolio/<slug>/interview-notes.md |

4. 확정된 항목만 해당 `portfolio/<slug>/interview-notes.md` **파일 끝에**
   `## 실전 환류 ([YYYY-MM-DD], [회사])` 섹션으로 append합니다 —
   스킬의 append 형식(질문 + 당시 답변 + 자가 판정 + 보강 필요 포인트)을 준수하고,
   **기존 섹션의 수정·삭제·재정렬은 절대 하지 않습니다**
5. 매칭 slug가 없는 경험 질문은 append하지 않고 보고서에만 기록합니다
   (**새 slug를 만들지 않습니다** — slug 생성은 `/portfolio` Phase 1의 책임)
6. 기술 개념 질문·컬처핏 질문은 파일 환류 없이 보고서의 갭 분류 표 /
   '다음 모의면접 참고' 섹션에 기록합니다 (질문지·모의면접 세션 파일을 직접 수정하지 않습니다)

### 5단계: 보고서 완성 및 안내

1. **반복 취약 영역 판정**: 스킬 규칙에 따라 이전 retro들과 대조 —
   같은 영역(질문은행 영역 / portfolio slug / 컬처핏 카테고리)에서 "버벅임/못 답함"이
   retro 2회 이상(이번 포함) 등장하면 반복 취약. 이전 retro가 없으면
   "첫 회고 — 다음 회고부터 추적"을 명기합니다
2. **액션 아이템 생성**: 스킬의 고정 매핑을 적용합니다 — 모든 항목은
   **기존 커맨드와 연결된 실행 가능 항목**이어야 하며 커맨드 원문(인자 포함)을 그대로 기재합니다:
   - B/C 분류 존재 → `/create-questionnaire` 재생성 (+ `/evaluate` 검증)
   - 경험 질문 환류 발생 → `/portfolio` 해당 slug 재실행 (Phase 2가 환류 노트를 로드)
   - 컬처핏·임원 질문 버벅임/못 답함 → `/mock-interview --type culture|executive` 재응시
   - A 분류인데 버벅임/못 답함 → 전달 갭: 질문지 답변 재작성 후 `/evaluate` 또는 `/mock-interview` 리허설
   - 반복 취약 영역과 연결된 항목은 **최상단** 배치 + "N회 반복" 표기
3. 보고서 나머지 섹션(갭 분류 표·반복 취약 영역·환류 기록·다음 모의면접 참고·액션 아이템)을
   스킬의 출력 형식대로 완성해 저장합니다
4. 사용자에게 핵심 요약을 표시합니다: 갭 분류 집계(A/B/C 건수·리허설 적중률),
   반복 취약 영역, 액션 아이템 상위 1~2개, 환류된 파일 목록

---

## 완료 후 안내

- 이 retro 보고서는 다음 `/interview-retro` 실행 시 **반복 취약 영역 대조 자산**으로 자동 활용됩니다
- `interview-notes.md`에 환류된 실전 질문은 `/portfolio` 재실행 시 Phase 2가 보강 재료로 로드합니다
- 액션 아이템의 커맨드를 실행한 뒤 다음 면접을 치르고, 면접 직후 다시 `/interview-retro`를
  실행하는 것이 이 커맨드의 설계된 사용 주기입니다 (기억이 증발하기 전에)
