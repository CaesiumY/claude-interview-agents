# 오퍼 검토·연봉 협상 대비 명령

받은 오퍼를 구조화해 비교하고, 자산 파일의 검증된 성과를 협상 근거 문장으로 변환하고,
인사담당자 상대역과 협상 대화를 리허설합니다.
**에이전트 없이 커맨드가 직접 진행하는 라이트 버전**입니다 (정식 채점 없음).

## 범위 제한 — 반드시 준수

이 커맨드는 아래 3가지**만** 수행합니다:

| # | 하는 일 | 하지 않는 일 |
|---|---------|------------|
| ① | 오퍼 조건 구조화 비교 (수치 비교·트레이드오프 서술) | 어느 오퍼가 나은지 우열 단정 |
| ② | 검증된 성과 수치 → 협상 근거 문장 변환 | 자산 파일에 없는 수치·성과 창작 |
| ③ | 협상 대화 리허설 + 체크리스트 기반 피드백 요약 | 리허설의 점수화 (정식 채점) |

**시장 연봉가 추정 금지 (Binary 위반 금지)**: "3년차 프론트엔드면 시장에서 X만원은 받아야 한다"류의
문장을 **어떤 형태로도 생성하지 않습니다**. 근거 데이터 없이 생성된 시장가는 환각이며,
잘못된 앵커가 되어 협상을 망칩니다. 시장 데이터 확인(채용 플랫폼 공개 연봉, 지인 레퍼런스 등)은
**사용자 몫**입니다 — 이 사실을 산출물 파일 상단에도 반드시 명시합니다.
사용자가 "이 금액이 시장에서 적정한가요?"라고 물어도 금액을 추정하지 않고 이 규칙을 안내합니다.

## 사용법
```
/offer resumes/[이름].md
```

> **설계 사유**: 이 커맨드는 인터랙티브 세션(협상 리허설)이 중심이라 서브에이전트 없이
> 메인 루프가 직접 진행합니다 (서브에이전트는 사용자와 대화할 수 없음).

## 실행 절차

> **경로 규칙**: 입력 파일과 산출물 경로는 모두 **커맨드를 실행한 현재 작업 디렉토리 기준**입니다.

### 1단계: 준비

1. 이력서 파일 읽기

   **예외 처리**:
   - **파일 미발견**: 이력서 파일이 없거나 경로가 틀리면 `resumes/` 폴더의 파일 목록을 보여주고
     사용자에게 경로 확인을 요청한 뒤 **중단**합니다. 임의로 추측해 진행하지 않습니다.
   - **[이름] 결정 규칙**: 산출물 파일명의 `[이름]`은 입력 이력서 파일의
     **베이스네임(확장자 제외)을 그대로** 사용합니다 — `commands/create-questionnaire.md`의
     '[이름] 결정 규칙'과 동일합니다 (예: `resumes/kim.md` → `resumes/kim_offer_[YYYY-MM-DD].md`)

2. 협상 카드 재료 탐색 (Glob) — 존재하는 파일만 읽습니다 (**읽기 전용**, 수정 금지):
   - `portfolio/*/final.md`
   - `resumes/[이름]_review.md`
3. `${CLAUDE_PLUGIN_ROOT}/skills/offer-review/SKILL.md` 읽기
4. AskUserQuestion(**multiSelect**)으로 이번 실행에서 진행할 모드를 선택받습니다:
   - **오퍼 비교** — "받은 오퍼 조건을 총보상 4분류로 분해해 비교표를 만듭니다"
   - **협상 카드 만들기** — "이력서·포트폴리오의 검증된 성과를 협상 근거 문장으로 변환합니다"
   - **협상 리허설** — "인사담당자 상대역과 협상 대화를 한 턴씩 연습합니다"
5. **민감 정보 안내 출력**: "오퍼 금액은 민감 정보입니다. 이 커맨드의 산출물은
   로컬 파일(`resumes/`)에만 저장되며, 커맨드가 외부로 전송하지 않습니다."
6. 산출물 파일 생성: `resumes/[이름]_offer_[YYYY-MM-DD].md` (UTF-8) —
   헤더에 범위 제한·민감 정보 문구를 포함합니다 (아래 '산출물 파일 형식' 참조)
   - **기존 파일 처리**: 동일 경로에 파일이 이미 존재하면 파일명 끝에 `_2`, `_3` … 순번을 붙여
     새 파일을 생성합니다 (오퍼 기록은 협상 이력 자산이므로 덮어쓰지 않습니다)

**모드 실행 순서**: 복수 선택 시 반드시 **오퍼 비교 → 협상 카드 → 협상 리허설** 순서로 진행합니다
(리허설의 앵커 연출은 오퍼 비교에서 수집한 실제 조건을, 근거 발화 관찰은 협상 카드를 재료로 사용).

### 2단계: 모드 A — 오퍼 비교 (선택 시)

1. 비교할 회사 수를 확인한 뒤, **회사별로** 스킬의 '오퍼 비교 항목 정의(총보상 분해)' 표를 따라
   한 분류씩 인터랙티브로 수집합니다:
   - 확정 현금: 기본급 · 계약연봉 (포괄임금 여부)
   - 조건부 현금: 사이닝 보너스 (상환 조건 포함) · 성과급 (지급 조건·실지급률)
   - 주식: 스톡옵션/RSU (수량·행사가·베스팅·클리프)
   - 비금전: 복지 · 근무 형태 · 성장 요인
2. **모르거나 아직 못 받은 항목은 "미확인"으로 기록**합니다 — 값을 추측하거나 창작하지 않습니다.
   수치 창작 금지 원칙은 성과 수치뿐 아니라 오퍼 조건에도 동일하게 적용됩니다
3. 수집이 끝나면 산출물 파일에 저장합니다:
   - **구조화 비교표**: 총보상 4분류 × 회사별 열
   - **항목별 관찰**: 스킬의 '관찰 작성 규칙' 준수 — 수치 차이와 트레이드오프 서술만 작성하고
     **우열 단정("A사가 낫다")과 시장가 언급은 금지**합니다.
     "미확인" 항목은 "오퍼 레터/인사팀에 확인 필요"로 표기합니다

**예외 처리 — 회사가 1곳**: 비교 대신 해당 오퍼의 총보상 분해표 + 미확인 항목 체크리스트를
생성합니다 (단일 오퍼도 구조화 자체가 협상 준비물입니다).

### 3단계: 모드 B — 협상 카드 만들기 (선택 시)

1. 1단계에서 읽은 재료 파일(이력서 · `portfolio/*/final.md` · `resumes/[이름]_review.md`)에서
   **검증된 정량 성과(PSR의 R)**를 추출합니다
2. 각 성과를 스킬의 '협상 카드 변환 규칙'에 따라 **협상 근거 문장으로 변환**합니다
   (예: "초기 로딩 3.2초→1.9초 개선" → "성능 개선으로 이탈률에 직접 기여한 경험이
   연봉 산정에 반영되기를 기대합니다") — 성과 유형(성능/장애 대응/생산성/협업)별 패턴 적용
3. 카드마다 **성과 원문과 출처 파일 경로를 병기**합니다 (협상 중 근거를 요구받으면 즉시 답할 수 있도록)
4. **수치 창작 금지 (Binary 판정)**: 카드의 모든 수치는 자산 파일 원문에 있는 것만 사용합니다.
   원문에 없는 수치가 카드에 하나라도 있으면 해당 카드를 폐기합니다.
   `[확인 필요]` 플레이스홀더가 붙은 수치도 카드로 만들지 않습니다 (검증 안 된 수치로 협상하면 역효과)

**예외 처리 — 정량 성과 0개**: 재료 파일 어디에도 정량 성과가 없으면 카드를 창작하지 않고
**중단·고지**합니다: "협상 카드의 재료가 될 검증된 수치가 없습니다.
`/portfolio`의 Phase 2(정량 질문)로 수치를 확보한 뒤 다시 실행하세요."

### 4단계: 모드 C — 협상 리허설 (선택 시)

`/mock-interview`의 세션 프로토콜 컨벤션을 준용합니다.

**세션 시작 오리엔테이션 (세션당 1회만 출력):**
리허설 시작 전에 다음을 안내합니다 — 상대역은 인사담당자 페르소나라는 것,
종료 방법("그만" 또는 "종료"라고 말하면 리허설이 끝납니다), 피드백은 종료 후
체크리스트 요약으로 제공된다는 것. 단, **준비된 협상 상황 목록은 미리 공개하지 않습니다.**

**대상 오퍼 설정**: 모드 A를 실행했으면 수집된 회사 중 리허설 대상 1곳을 선택받습니다.
모드 A 없이 리허설만 실행하는 경우 최소 정보(회사명 · 제시받은 조건)만 간단히 수집합니다.

**진행 프로토콜 — 반드시 준수:**

1. 상대역은 스킬의 '인사담당자 페르소나'를 유지하고, '현실적 협상 상황 5종'
   (앵커링·마감 압박·경쟁 오퍼 확인·복지로 우회·재고 불가 통보)에서 **3~4종을 선별**해 연출합니다
   — **앵커링("저희 기준상 이 정도가 최선입니다")은 항상 포함**, 침묵 연출은 스킬의 보조 기법 규칙 준수
2. **한 번에 한 턴씩** 진행합니다. 남은 상황 목록을 미리 보여주지 않습니다
3. 답변을 받으면 **즉시 산출물 파일의 '협상 리허설 기록' 섹션에 append**합니다 (중단돼도 기록 보존)
4. **상대역의 금액 창작 금지**: 앵커링·조정 연출은 사용자가 입력한 **실제 제시 조건만** 기준으로
   합니다. 상대역이 임의의 구체 금액을 새로 만들지 않으며, 조정 여지를 표현할 때는
   "제시안 기준 소폭"과 같은 비수치 화법을 사용합니다
5. **세션 중 피드백 금지**: 답변에 대한 평가, 칭찬, 교정을 하지 않습니다.
   실제 협상 상대처럼 중립적으로 다음 턴을 이어갑니다 (실전감 유지)
6. **메타 발화 처리** — 인사담당자 페르소나를 유지한 채 대응합니다:
   - 재요청("방금 뭐라고 하셨죠?") → 직전 발화를 다시 진술합니다
   - 스킵 요청("이 상황은 넘어갈게요") → 기록에 `(스킵)`을 남기고 다음 상황으로 넘어갑니다
   - 세션 중 피드백 요구("방금 대응 어땠어요?") → "피드백은 리허설 종료 후
     체크리스트 요약으로 제공됩니다"라고 정중히 안내하고 협상 톤을 유지합니다
7. **종료 조건** — 아래 둘 중 하나를 만족하면 피드백 작성으로 전환합니다:
   - (a) 선별한 상황을 모두 소진하면 클로징 발화("오늘 논의는 여기까지 하고, 정리해서
     다시 연락드리겠습니다")로 마무리합니다
   - (b) 사용자가 "그만", "종료"라고 하면 즉시 종료합니다 (클로징 생략)

**종료 후 — 체크리스트 기반 피드백 요약:**

이 피드백은 **정식 채점이 아닙니다** (점수·등급·합불 판정 없음). 평가 에이전트를 호출하지 않고
스킬의 '협상 체크리스트' 4항목(근거 제시력/앵커 대응/양보 순서/감정 관리)을 기준으로
**커맨드가 직접** 요약을 작성해 산출물 파일에 저장합니다:

- 항목별로 **세션 기록의 실제 발화를 인용**해 관찰 근거를 제시합니다
- 관찰할 발화가 없었던 항목은 "관찰 기회 없음"으로 표기합니다 (억지 피드백 금지)
- 개선 제안은 항목당 1개, 스킬의 '좋은 대응 예시' 방향으로 작성합니다

### 5단계: 결과 저장 및 안내

1. 실행한 모드의 섹션만 산출물 파일에 남았는지 확인합니다 — **실행하지 않은 모드의
   빈 섹션은 만들지 않습니다**
2. 사용자에게 파일 경로와 핵심 내용을 요약해 안내합니다
3. 마무리 안내 (매 실행 동일):
   - "시장 연봉가 확인은 이 커맨드의 범위 밖입니다. 채용 플랫폼 공개 연봉·지인 레퍼런스 등
     시장 데이터는 직접 확인하세요."
   - "합의된 조건은 반드시 서면(이메일·처우 통지서)으로 확인하세요."
   - 스킬의 '3년차 협상에서 흔한 실수' 표를 협상 전에 한 번 읽어볼 것을 권합니다

## 1. 오퍼 비교

### 비교표
| 분류 | 항목 | A사 | B사 |
|------|------|-----|-----|
| 확정 현금 | 기본급 | ... | ... |
| 확정 현금 | 계약연봉 | ... | 미확인 |
| 조건부 현금 | 사이닝 보너스 (상환 조건) | ... | ... |
| ... | ... | ... | ... |

### 항목별 관찰
